# Documentation Structure

This guide defines the structural patterns for Vantle documentation.

## Rendering Framework

Documentation pages are generated by a Rust rendering framework. Source files (`.document.rs`) use a builder DSL to construct pages, which are then rendered to static HTML.

The API may change â€” this guide describes the content patterns, not the exact API.

### Pipeline

1. `.document.rs` source defines page content via builder closures
2. `bazel run` compiles and executes the generator
3. Generator renders HTML with nav, sidebar, content, and auto-generated TOC
4. `publish` tool copies generated files to the workspace

### Template

Every page gets a standard template with:
- Sticky navigation bar with dropdown menus
- Three-column layout: sidebar | main content | table of contents
- Auto-generated TOC from heading hierarchy
- Footer with copyright
- Theme toggle (light/dark)
- WASM-powered interactivity (scroll tracking, copy buttons)

## Content Sections

Documentation pages follow this content ordering within the main area:

1. Title and subtitle
2. Introduction paragraph explaining purpose
3. Theory or concepts (if needed)
4. Primary feature with usage example (Invoke/Usage)
5. Secondary features
6. Examples

## Required Content

Every documentation page must include:

1. **Title**: page heading
2. **Introduction**: one paragraph explaining what this component does
3. **Primary Feature**: main functionality with usage example

## Optional Content

Include when applicable:

- **Arguments/Options**: for CLI tools or configurable components
- **Examples**: additional usage examples beyond basic usage
- **Theory**: conceptual background for complex systems
- **Features**: feature list for multi-feature components

## Link Conventions

### Relative Paths

Generated pages use relative paths for internal navigation. The `data-root` attribute on `<html>` tracks depth for correct link resolution.

### Standard Links

Every page should link to:

| Link | Purpose |
|------|---------|
| Parent context | Root documentation (Vantle or Molten) |
| Module | Dependencies |
| License | Licensing |

## Completeness Checklist

- [ ] Title and subtitle present
- [ ] Navigation links to parent and related docs
- [ ] Introduction paragraph explaining purpose
- [ ] At least one usage example
- [ ] All code examples compile and work
- [ ] All internal links resolve
- [ ] No orphaned sections without content
