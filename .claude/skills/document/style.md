# Vantle Documentation Style

This guide defines the visual style for Vantle documentation. All styling is handled by the CSS generation system — no hand-written CSS or inline styles.

## Design System

Styling is generated by the Rust framework (`system/generation/web/style.rs`) and output as `vantle.css`. The design system uses CSS custom properties for consistent proportions and theming.

## Golden Ratio

The golden ratio (phi = 1.618) governs all proportions through CSS variables:

| Variable | Scale | Value |
|----------|-------|-------|
| `--scale-n2` | phi^-2 | 0.382rem |
| `--scale-n1` | phi^-1 | 0.618rem |
| `--scale-0` | phi^0 | 1rem |
| `--scale-1` | phi^1 | 1.618rem |
| `--scale-2` | phi^2 | 2.618rem |
| `--scale-3` | phi^3 | 4.236rem |

Apply to: spacing, sizing, typography, aspect ratios, animation timing.

## Themes

Light and dark themes are supported via `data-theme` attribute on `<html>`. Colors are defined in `component/design/palette.rs` as paired light/dark values. The theme toggle persists selection in localStorage.

## Typography

Heading sizes follow the golden ratio scale:
- h1: `--scale-3`
- h2: `--scale-2`
- h3: `--scale-1`
- h4–h5: `--scale-0`

Body text uses system fonts. Code uses monospace fonts (SF Mono, Fira Code, Cascadia Code).

## Code Highlighting

Syntax highlighting is generated by `system/generation/web/html/highlight/` modules with language-specific highlighters (Rust, Molten, Bash, JSON, Starlark, Python). Output uses semantic CSS classes:

- `syntax-keyword`, `syntax-entity`, `syntax-string`
- `syntax-comment`, `syntax-constant`, `syntax-storage`
- `syntax-punctuation`, `syntax-variable`, `syntax-function`
- `syntax-operator`, `syntax-macro`

## Layout

Three-column grid layout:
- **Sidebar** (left): page navigation with current page indicator
- **Main** (center): page content
- **Outline** (right): auto-generated table of contents with scroll tracking

Responsive breakpoints collapse columns at 1280px and 1024px.

## Content Style

- Use bold for key terms on first use
- Use italic for emphasis
- Use inline code for commands, paths, function names
- Use horizontal rules between major sections
- Use blockquotes for notes or asides
- Use tables for structured reference information
- Use definition lists for terminology

## Style Checklist

- [ ] Content uses proper heading hierarchy (h1 > h2 > h3)
- [ ] Code blocks specify language for syntax highlighting
- [ ] Tables used for structured data where appropriate
- [ ] Internal links use correct relative paths
- [ ] Sections separated by horizontal rules
- [ ] Copyright footer present
